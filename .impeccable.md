## Design Context

### Users
oorouter is used by developers and power users running a local ChatGPT/Codex OAuth proxy. They are usually checking whether the local proxy is up, which models are exposed, whether calls are flowing, and whether token usage is being recorded.

### Brand Personality
Quiet, precise, operational. The interface should feel like a trustworthy local control room: compact, fast to scan, and calm under failure.

### Aesthetic Direction
Use a restrained desktop utility aesthetic rather than a marketing dashboard. Prioritize dense but readable lists, crisp status surfaces, tinted neutral colors, and strong typographic alignment. Support both light and dark modes without neon or gradient-heavy styling.

### Design Principles
- Put runtime truth first: server state, port, model, status, and usage must be visible without decorative noise.
- Use cards only for distinct tools or comparable repeated items; prefer tables, rows, dividers, and bands for operational data.
- Keep controls compact and predictable, with clear disabled states when the browser preview cannot control the Tauri backend.
- Make logs, models, and usage feel like one system by sharing spacing, table density, badges, and status color semantics.
- Use subtle contrast and tinted neutrals; avoid generic AI dashboard gradients, oversized hero sections, and decorative blobs.
